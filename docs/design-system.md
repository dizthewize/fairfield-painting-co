## Aesthetic Direction

**Committed style: Editorial Craftsmanship.** This design commits to an editorial, warm-minimalist aesthetic because the painting trade is a *craft* business, not an emergency service — the visual tone should communicate taste, care, and a calm eye for color rather than urgency or hazard-signal branding.

**Personality:** Confident and restrained. Fraunces headlines carry visual weight on their own, so supporting UI stays quiet — thin horizontal rules instead of boxed cards where possible, generous whitespace, photography-first sections. Amber is used sparingly as an earned accent (primary CTAs, link states, small highlights); charcoal carries the structural weight.

**Border radius:** 6px on interactive elements, 10px on cards, 16px on hero frames. Never pill-shaped buttons — looks consumer-app, wrong category.

**Shadows:** Subtle. No glow effects, no chunky drop-shadows. `shadow-md` on cards that lift on hover. Hero imagery uses a soft vignette rather than a heavy dark overlay.

**Motion posture:** Moderate. One signature entrance animation — hero headline fades in with a 30ms staggered letter reveal using Fraunces's optical sizing axis (subtle weight shift). Hover states are quick (120ms) and restrained (2px translateY on cards; color shift on buttons). No parallax. No scroll-triggered cascades.

**Layout principles:** Editorial columns over rigid card grids. Hero is asymmetric (8-column text + 4-column embedded form on desktop). Services grid uses photography-forward cards with the service name set as large Fraunces type — feels like a design magazine feature rather than a contractor landing page. Full-bleed exterior-painting hero image on service pages. Color-palette swatches appear as a signature visual device on the homepage (a thin horizontal band showing the house's proposed palette — a unique painting-trade visual).

**Anti-patterns explicitly avoided:**
1. **No purple/gradient hero backgrounds** — wrong emotional register for a craft trade.
2. **No Inter, Roboto, or Poppins** — overdone and generic; Fraunces + Manrope gives us a distinctive fingerprint.
3. **No stock photos of painters in white coveralls giving thumbs-up** — use process photography, color-swatch-on-wall details, hands-with-brush tight shots.
4. **No pill-shaped buttons, no neon CTAs** — signals consumer app, not craft trade.
5. **No "card everything"** — let sections breathe; use horizontal rules and typography weight to create hierarchy instead of boxing every piece of content.

### Conversion

**Button**
- Shared: `font-family: var(--font-body)`, `font-weight: 600`, `letter-spacing: 0.01em`, `border-radius: var(--radius-md)`, transitions all 120ms ease
- **Primary:** `bg-[--color-primary] text-white` → hover: `bg-[--color-primary-dark]` + `translateY(-1px)` + shadow-sm
- **Secondary:** `border-2 border-[--color-accent] text-[--color-accent] bg-transparent` → hover: `bg-[--color-accent] text-[--color-text-inverse]`
- **Ghost:** `text-[--color-primary] underline underline-offset-4 decoration-1` → hover: `decoration-2`
- Sizes:
  - sm — `text-sm` / padding 10px 16px / height 36px
  - md — `text-base` / padding 12px 22px / height 44px (default)
  - lg — `text-lg` / padding 16px 28px / height 52px
- Focus ring: 3px `--color-primary` at 30% opacity, offset 2px
- Never pill-shaped

**LeadCaptureForm**

- Container: `--color-surface` background, `--radius-lg`, 24px padding on desktop / 20px mobile, shadow-md
- Form heading (optional): Fraunces 24px, "Request Your Free Estimate"
- Fields in order:
  1. Name (text, required)
  2. Phone (tel, required) — with `inputmode="tel"` + `autocomplete="tel"`
  3. Email (email, required) — with `autocomplete="email"`
  4. Service Type (select, required) — options: Interior Painting, Exterior Painting, Cabinet Refinishing, Deck Staining, Drywall Repair, Not sure yet

## Accessibility Checklist

- [ ] Color contrast: all text/background combos meet WCAG AA (verified — see Color System)
- [ ] Touch targets: all interactive elements ≥ 44×44px (buttons default to 44px height)
- [ ] Focus indicators: visible 3px amber ring on all focusable elements
- [ ] Phone links: all `<a href="tel:...">`, never plain text
- [ ] Form labels: all form fields have associated `<label>` (explicit `for`/`id`)
- [ ] Images: all `<img>` have descriptive `alt` attributes (decorative images use `alt=""`)
- [ ] Accordion: `aria-expanded`, `aria-controls` on all FAQ items; keyboard Enter/Space toggles
- [ ] Mobile sticky bar: not present on desktop (`@media min-width: 768px { display: none }`)
- [ ] Headings: single h1 per page, no skipped heading levels
- [ ] Skip link: "Skip to main content" as first focusable element, hidden until focused
- [ ] Reduced motion: respect `prefers-reduced-motion` — disable hero stagger animation, all transitions become instant
- [ ] Forms announce errors via `aria-live="polite"` region
- [ ] Dark mode: out of scope for v1 (site is warm-light by design); can revisit if requested
